# PROV-O lineage mapping

The TrustGraph projection — `poc_v1/ontology/trustgraph_ontology.json`,
generated by `scripts/generate_trustgraph_ontology.py` — maps the ontology's
experiment-lineage nodes and edges onto the [W3C PROV-O][provo] vocabulary.
TrustGraph's explainability layer expects provenance as a typed graph
conforming to PROV-O; this mapping is the contract that makes the
Subconscious ontology legible to it.

Every PROV-O term below traces to a field that already exists on the schema
(`poc_v1/ontology/schema.py`, v1.6.0) — no `prov:` classes or edges are
added to the ontology itself. The mapping lives only in the projection.

## Classes

| Ontology node | PROV-O class | Rationale |
|---|---|---|
| `ExperimentRun` | `prov:Activity` | a bounded experiment execution with a start/end |
| `Estimate` | `prov:Entity` | a value produced by an Activity |
| `Evidence` | `prov:Entity` | a source-grounded record consumed/derived from |

## Edges

| Ontology edge | PROV-O property | Direction |
|---|---|---|
| `CONSUMED` | `prov:used` | `ExperimentRun` → ontology context node |
| `PRODUCED` | `prov:wasGeneratedBy` | `ExperimentRun` → `Estimate` |
| `SUPPORTS` | `prov:wasDerivedFrom` | `Evidence` → any node |

## Agents and timestamps

PROV-O agent attribution and activity timing come from node fields, not
edges:

| Schema field | PROV-O |
|---|---|
| `agent_type` + `agent_id` (Evidence, Estimate, ExperimentRun) | `prov:Agent` — via `prov:wasAttributedTo` (Entities) / `prov:wasAssociatedWith` (Activities) |
| `ExperimentRun.started_at` / `completed_at` | `prov:startedAtTime` / `prov:endedAtTime` |
| `Estimate.estimated_at` | `prov:generatedAtTime` |
| `Evidence.generated_at` | `prov:generatedAtTime` |

The `agent_*` fields and `Evidence.generated_at` were added in schema v1.6.0
(see `poc_v1/MIGRATION.md`); `started_at` / `completed_at` / `estimated_at`
pre-date it.

## Deliberately out of scope

The projection is a *minimal conformance projection* (issue #71). It carries
class IRIs, property types, required-ness, predicate domain/range, and this
PROV-O mapping as plain JSON. Formal OWL axiom serialization
(`owl:minCardinality`, `owl:oneOf`, `rdfs:domain`/`range`) and triple-clean
RDF/Turtle output are **deferred** until a TrustGraph backend actually
ingests the artifact — TrustGraph is a conformance target we project toward,
not a backend we migrate to (umbrella issue #70).

[provo]: https://www.w3.org/TR/prov-o/
