market-ontology Architecture

Canonical Pydantic schema for the Subconscious knowledge graph • schema.py is the single source of truth • SCHEMA_VERSION 1.6.0

Source of truth — schema.py

  • • 14 NODE_MODELS, 18 EDGE_MODELS, SCHEMA_VERSION 1.6.0 — the only definition of graph shape.
  • • graphiti_views.py derives ENTITY_TYPES / EDGE_TYPES / EDGE_TYPE_MAP — add a node, it propagates.
  • • identity.py and iri.py are schema-decoupled boundary serializers (slugs, RDF IRIs).
  • • All *.json / *.ttl under poc_v1/ontology/ are generated — never hand-edited.

causal_dag_v1 — the causal peer

  • • poc_v1 is a property graph of types — structural edges (HAS_ATTRIBUTE, FROM, TO).
  • • causal_dag_v1 holds the causal layer: CAUSES, MEDIATES, MODERATES, CONFOUNDED_BY.
  • • Own Pydantic models + NetworkX acyclicity check in validate.py.
  • • Ontology = context; causal hypotheses = result. Kept structurally separate.

Consumers & CI gate

  • • Consumers pip install the package and import the public modules directly.
  • • spice-harvester validates every kg_seed line against the schema at write time.
  • • twenty, burn-substrate, native-sizzle consume the derived projections / views.
  • • main is gated by ci.yml + symphony-gate.yml; Python 3.11 is the CI baseline.
